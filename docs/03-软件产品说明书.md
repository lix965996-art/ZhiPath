> **项目名称**：ZhiPath（智径）—— 基于大模型的个性化资源生成与学习多智能体系统
> **赛题**：第十五届中国软件杯大赛 A 组赛题 A3
> **文档版本**：V2.0
> **编写日期**：2026-06-15

## 1 产品概述

### 1.1 一句话讲清

ZhiPath 是一款面向高等教育场景的个性化学习多智能体系统——学生开口说话就能被系统自动建画像、生成多模态学习资源、追踪掌握度并动态调整学习路径，教师可查看班级聚合数据。

### 1.2 三句话讲清

1. 学生用自然语言提出学习目标后，系统从 7 个维度自动构建学习者画像，不需要填表。
2. 多个 AI 智能体协作生成 8 类多模态资源（测验、闪卡、思维导图、代码实操、Mermaid 图表、知识图谱、案例研究、动画讲解音频），并用 BKT + DKT + IRT + FSRS-4.5 四套算法持续追踪学情。
3. Auto-Tutor 自动闭环：目标诊断 → 资源生成 → 自评测验 → 画像更新 → 路径重规划，全程流式可视。

### 1.3 产品理念

ZhiPath 的核心理念是"因材施教，算法驱动"。区别于传统题库和通用 AI 对话，ZhiPath 把教育数据挖掘算法（知识追踪、项目反应理论、间隔重复）作为学情建模的骨架，把大模型作为内容生成的引擎，让每一次学习行为都被量化、追踪、反馈。

### 1.4 与同类产品的差异

| 对比维度 | 传统题库系统 | 通用 AI 对话 | ZhiPath |
|---------|------------|------------|---------|
| 画像构建 | 表单填写 | 无画像 | 对话自动抽取 7 维画像 |
| 资源生成 | 预制题库 | 单模型一次性输出 | 多智能体并行生成 8 类资源 |
| 学情追踪 | 记录错题 | 不追踪 | BKT/DKT/IRT/FSRS 四算法 |
| 学习路径 | 固定顺序 | 无路径 | 基于掌握度动态重规划 |
| 防幻觉 | 不涉及 | 无约束 | 引用标注 + 内容安全护栏 |
| 可视化 | 成绩报表 | 无 | 多智能体调用链实时可视化 |

## 4 快速上手

### 4.1 环境准备

| 依赖 | 版本要求 |
|------|---------|
| Node.js | 18+ 或 20+ |
| Python | 3.12+ |
| PostgreSQL | 16+（需 pgvector 扩展） |
| Docker（可选） | 24+ |

### 4.2 方式一：Docker Compose 一键启动

适合快速演示。

```bash
docker compose up --build
```

启动后访问：

- 前端：http://localhost:3000
- 后端：http://localhost:8000
- API 健康检查：http://localhost:8000/health

Docker Compose 包含三个服务：

| 服务 | 镜像 | 端口 |
|------|------|------|
| postgres | pgvector/pgvector:pg16 | 5432 |
| backend | 项目构建 | 8000 |
| frontend | 项目构建 | 3000 |

数据库默认连接：`postgresql+asyncpg://zhipath:zhipath@postgres:5432/zhipath`

### 4.3 方式二：本地启动

适合开发调试。

**步骤 1：配置环境变量**

```bash
cp .env.example .env
```

编辑 `.env`，至少配置 `DATABASE_URL`。真实 LLM / TTS Key 不建议写入仓库，优先在前端设置面板中填写。

**步骤 2：启动后端**

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate    # Linux/Mac
pip install -r requirements.txt
python main.py
```

后端启动后会自动初始化数据库表和 RAG 管线。

**步骤 3：启动前端**

```bash
cd frontend
npm install
npm run dev
```

**步骤 4：打开页面**

```
http://localhost:3000
```

### 4.4 首次使用三步

1. **配置 API Key**：进入对话页，按引导或在右上角设置中填入 LLM 凭据（可填多套并测试连通性）。
2. **（可选）填充演示数据**：在工作台点击一键填充，注入 408 知识库和示例画像数据。
3. **选择角色**：学生 / 教师 / 演示，开始体验。

### 4.5（可选）启用讯飞能力

赛题要求优先选用科大讯飞相关工具。系统已接入：

| 讯飞能力 | 用途 | 配置方式 |
|---------|------|---------|
| 讯飞星火 LLM | OpenAI 兼容接口，作为备选 LLM | 前端设置页或 `XF_SPARK_API_PASSWORD` |
| 讯飞超拟人 TTS | 讲义音频化（WebSocket 协议） | 前端设置页或 `XF_TTS_APPID` / `XF_TTS_API_KEY` / `XF_TTS_API_SECRET` |

未配置讯飞凭据时，相关功能自动降级，不影响主流程。

## 5 分功能使用指南

### 5.1 对话工作台（/chat）

核心交互入口。用户输入学习目标或问题后，系统自动路由到对应能力（目标诊断、学习路径、资源生成、自动闭环、辩论、智能辅导等）。右侧面板实时展示多智能体调用链路（Agent 间消息、工具调用、阶段进度），让学习过程透明可追溯。

### 5.2 学习者画像（/profile）

展示从对话中自动抽取的 7 维画像：学习目标、当前水平、关注主题、薄弱知识点、学习偏好、时间约束、最近意图。每个维度附带证据链（来源对话片段），可验证画像不是凭空生成的。

### 5.3 课程知识库（/knowledge）

内置 408 四科知识切片，支持关键词检索和向量语义检索。点击知识点可查看拓扑结构（前置依赖关系）和关联知识，帮助定位薄弱环节。

### 5.4 资源工坊（/resources）

查看系统生成的全部学习资源，按类型分类浏览：测验题、闪卡（FSRS 复习卡片）、思维导图（Mermaid 渲染）、代码实操（Code Lab 沙箱）、Mermaid 流程图、知识图谱片段、案例研究。资源包支持导出。

### 5.5 学习路径（/path）

阶段式学习路线，每个阶段包含推荐资源、练习任务和完成标准。掌握度变化后路径自动重规划。

### 5.6 学习仪表盘（/dashboard）

知识图谱可视化（节点 = 知识点，边 = 依赖关系）、掌握度热力图（BKT 概率映射颜色深浅）、FSRS 复习日历（到期卡片提示）。

### 5.7 系统总览（/overview）

面向教师和评审，展示系统能力清单、核心流程图、工程结构和统计数据。

## 6 系统配置

### 6.1 环境变量

| 变量 | 必填 | 说明 |
|------|------|------|
| `DATABASE_URL` | 是 | PostgreSQL 连接串，Docker 默认 `postgresql+asyncpg://zhipath:zhipath@postgres:5432/zhipath` |
| `BACKEND_HOST` | 否 | 后端监听地址，默认 `0.0.0.0` |
| `BACKEND_PORT` | 否 | 后端端口，默认 `8000` |
| `FRONTEND_PORT` | 否 | 前端端口，默认 `3000` |
| `DEEPSEEK_API_KEY` | 否 | DeepSeek Key，可在前端设置页填写 |
| `DASHSCOPE_API_KEY` | 否 | 通义千问 Key |
| `SILICONFLOW_API_KEY` | 否 | 硅基流动 Key |
| `XF_SPARK_API_PASSWORD` | 否 | 讯飞星火 OpenAI 兼容接口凭据 |
| `XF_TTS_APPID` | 否 | 讯飞 TTS AppID |
| `XF_TTS_API_KEY` | 否 | 讯飞 TTS API Key |
| `XF_TTS_API_SECRET` | 否 | 讯飞 TTS API Secret |

### 6.2 用户级凭据配置

系统支持用户在前端设置页自行配置多套 LLM 凭据并测试连通性。凭据存储在数据库 `user_api_configs` 表中（JSONB 格式），不写入代码仓库。支持一次配置多套 Key，系统按优先级自动选择。

## 8 答辩演示脚本

### 8.1 开场（20 秒）

### 8.2 演示 1：对话式画像（30 秒）

1. 打开 /chat 工作台，输入"我想考研复习数据结构，基础一般，每天能学两小时"。

3. 切换到 /profile 页面，7 个维度已被自动填充，每个维度有证据链。

### 8.3 演示 2：多智能体资源生成（60 秒）

2. 系统路由到 resource_gen 能力，7 个 Agent 并行启动。
3. 右侧面板实时高亮各 Agent 状态（quiz/flashcard/mindmap/code_lab/mermaid/kg/case_study）。

## 9 常见问题

**Q1：不配置 LLM Key 能用吗？**

可以浏览界面和已有数据，但涉及内容生成的功能会降级或提示配置 Key。建议至少配置一个 DeepSeek Key。

**Q2：必须用 Docker 吗？**

不必须。Docker Compose 适合快速演示；本地启动适合开发调试。两种方式在文档中均有说明。

**Q3：pgvector 安装失败怎么办？**

Docker 方式自带 pgvector（pgvector/pgvector:pg16 镜像）。本地安装需先装 PostgreSQL 16，再编译 pgvector 扩展。如果完全无法安装 pgvector，系统会降级为纯 JSON 文件存储，主功能不受影响。

**Q4：讯飞能力不配置有影响吗？**

TTS 音频和星火 LLM 不可用，但系统会自动降级到其他已配置的 LLM，讲义以纯文本展示。主流程不受影响。

**Q5：Code Lab 代码沙箱安全吗？**

有双重防护：静态安全检查拦截危险函数（system/fork/exec/fopen 等）和未授权头文件；运行时限制 5 秒超时和 64KB 输出截断。适合教学演示，生产环境建议加强隔离。

**Q6：支持哪些 LLM？**

DeepSeek（V3/R1，主力）、通义千问、硅基流动多模型、讯飞星火，以及任何 OpenAI 兼容接口。前端设置页可配置多套凭据并测试连通性。

**Q7：数据库里存了什么？**

8 张表：用户、用户API配置、会话、消息、记忆、画像、文档、文档分片（含 768 维向量）。另有 18 个 JSON 文件目录存储 BKT 掌握度、FSRS 复习卡片、知识图谱等学习数据。
